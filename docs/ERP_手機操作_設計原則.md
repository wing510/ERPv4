## ERP 手機操作 — 設計原則（討論定案）

> **狀態**：定案（Phase 1 全站 CSS 已套用；Phase 2／3 尚未開始）  
> **日期**：2026-06-16  
> **定位**：手機以 **查詢 + 簡單登錄** 為主；**複雜建單不強求**與桌機等價

---

## 0. 一句話

**能用手機查、能辦小事；採購／出貨／Invoice 等複雜流程以電腦為主，手機不擋查詢、不硬塞多欄表單。**

---

## 1. 產品定位（已定案）

| 優先級 | 使用情境 | 代表模組 |
|--------|----------|----------|
| **高** | 查詢、確認、狀態 | Dashboard、倉庫庫存、Lot QA、追溯、AR 列表、Case 列表 |
| **中** | 簡單登錄（筆數少） | AR 收款、收貨幾筆、寄賣收回少量 LOT |
| **低** | 複雜建單、多列明細 | 採購、進口、銷售、出貨、Invoice、委外加工 |

手機版**不要求**完整取代桌機；複雜頁可顯示「建議使用電腦操作」類提示（日後選做）。

---

## 2. 現況基礎建設（改版前已有）

| 項目 | 說明 |
|------|------|
| `viewport` | `width=device-width, initial-scale=1` |
| ≤768px 導覽 | 側欄改 **漢堡抽屜** + 遮罩 |
| `form-grid-4` | ≤768px 改 **單欄** |
| `.table-wrapper` | 固定高度 + **橫向／縱向捲動** + `-webkit-overflow-scrolling:touch` |
| Toast | `max-width` 適應窄螢幕 |

**缺口**：`.table-wrap`（寄賣等模組）、表單底 **按鈕列並排**、表格內 **小按鈕** 觸控區偏小、部分 `min-width` 下拉過寬。

---

## 3. 全模組共通原則

### 3.1 版面

- **表單**：768px 以下一律單欄（沿用 `form-grid-4` 規則）。
- **寬表格**：外層必須可 **橫向捲動**；禁止靠縮小字體硬塞所有欄位。
- **列表**：優先 **少欄 + 點列展開**（Case 案件、結算／收回歷史）；避免依賴多顆 `btn-sm`。
- **主檔**（未來版型 A）：列表上、表單下；Load 後 **捲到表單**（見 `docs/ERP_主檔版型A_討論定案說明書.md`）。

### 3.2 觸控

- 主要按鈕（表單底 **建立／更新／清除**）：窄螢幕 **直向堆疊、全寬**，觸控高度約 **≥44px**。
- 表格內按鈕：手機上 **加大**（仍小於表單主按鈕），能點為準。
- **可點整列** 優於小按鈕（Case `cc-case-row`、歷史 `cc-history-row`）。

### 3.3 互動

- 切換模組後 **關閉側欄抽屜**（沿用 `navigate` 行為）。
- 避免 `window.prompt` 作為主要輸入（鍵盤擋畫面）；日後改頁內欄位或底部 sheet。
- 避免 `min-width:280px` 等固定寬下拉在窄螢幕溢出（新頁面禁止，舊頁逐步修）。

### 3.4 新頁面／改版檢查表

- [ ] 表單在 768px 是否單欄可讀？  
- [ ] 表格外層是否 `.table-wrapper` 或 `.table-wrap`？  
- [ ] 主操作是否不必點 `btn-sm`？  
- [ ] 表單底按鈕是否走 `.button-group`（吃全站手機樣式）？  
- [ ] 工具列按鈕（重新整理等）是否保持橫排（在 `card-body` 內層 flex，不受直向堆疊影響）？

## 5. 實作分階段

### Phase 1 — 全站 CSS（**已完成 2026-06-16**）

**檔案**：`index.html` 內 `@media (max-width:768px)` 及全域 `.table-wrap`

| 項目 | 內容 |
|------|------|
| 表單底 `.button-group` | `card-body` 直下按鈕列直向堆疊、全寬、min-height 44px |
| 例外 | `.mv-button-group`（庫存異動三鈕）、工具列內嵌 `button-group` 維持橫排 |
| 表格內按鈕 | `tbody button` / `.btn-sm` 加大觸控區 |
| `.table-wrap` | 與 `.table-wrapper` 相同橫向捲動 |
| 可點列 | `.cc-case-row` / `.cc-history-row` 列高略增 |

**驗收**：手機寬度下 Products 表單底三鈕直向全寬；寄賣表格外層可橫捲；Case 點列仍正常。

### Phase 2 — 高頻手機場景（未做）

- Dashboard、倉庫庫存、AR、Case：列表 **卡片化** 或強化點列詳情  
- 收貨、寄賣收回：明細 **一列一卡** 輸入  
- 作廢原因等：取代 `prompt` 為頁內表單  

### Phase 3 — 其餘模組（未做）

- 採購、出貨、Invoice：**唯讀查詢** + 必要時提示用電腦  
- 不追求手機完整編輯多列明細

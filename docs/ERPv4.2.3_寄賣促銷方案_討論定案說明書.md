## ERP v4.2.3 寄賣促銷方案 — 討論定案說明書

> **狀態**：開發中（MVP 程式已實作；須先執行 SQL）  
> **版本**：v4.2.3（寄賣模組 · 促銷方案）  
> **日期**：2026-06-16  
> **範圍**：寄賣模組新增 **促銷方案主檔**；**結算**時自動套用並預覽計價結果  
> **前置**：v4.2.2 寄賣案件（品項池、結算、收回、AR）已上線  
> **開發策略**：分階段（MVP 同品項固定價／折扣／買 N 送 M；進階累計與組合促銷後做）

---

## 0. 一句話總結（最重要）

**促銷在「結算」套用：池子出貨單價不動；結算量全扣未售；計價量依方案算錢產生 AR。結算畫面不逐列選促銷，改自動匹配 + 贈送／計價／小計預覽；僅衝突時在品項層手選。**

---

## 1. 目的（要解決的痛點）

- 店家活動（特價、折扣、**買三送一**）需反映在 **結算應收**，不能每次靠人工改 AR 或口頭約定。
- 品項池可能 **同產品、多筆出貨、不同單價**；促銷須可對帳、可追溯。
- 結算操作應 **少步驟**：操作員填「賣出量」，系統算促銷與金額。

---

## 2. 與現有系統的關係

| 保留 | 新增／調整 |
|------|------------|
| 品項池 `unit_price`（出貨合約價） | **不修改**；收回分配政策仍依池子單價 |
| 結算扣 `settled_qty`、產生 AR | 結算明細新增 **計價量／贈送量／套用方案** |
| `amount_system` / `amount_due` 財務口徑 | `amount_system` = Σ(計價量 × 結算單價) |
| AR 事後折讓 | 保留；**事後返利**類促銷可走 AR（Phase 3） |
| 分配政策（FIFO／高價先退） | 與促銷 **獨立**；促銷只影響結算計價 |

```text
出貨過帳 → 品項池（出貨單價）
                ↓
促銷方案主檔 ──→ 結算（settle_qty / billable_qty / free_qty）
                ↓
            應收 AR
```

## 4. 促銷方案主檔

### 4.1 表頭

| 欄位 | 必填 | 說明 |
|------|------|------|
| 方案編號 | 系統產生 | 如 `CP-…` |
| 方案名稱 | 是 | 對外／內部辨識 |
| 狀態 | 是 | 草稿／生效／結束 |
| 有效期起 | 是 | 含當日 |
| 有效期迄 | 是 | 含當日 |
| 適用範圍 | 是 | 見 4.2 |
| 備註 | 否 | |

### 4.2 適用範圍（優先序）

| 層級 | 說明 | 優先序 |
|------|------|--------|
| **指定寄賣案** | 僅該 `case_id` | 最高 |
| **指定客戶** | 該客戶下所有進行中案件 | 中 |
| **全站** | 所有寄賣結算（少用） | 最低 |

**衝突規則**（同品項同時命中多方案）：

1. 範圍窄者優先（案件 > 客戶 > 全站）  
2. 同層：**有效期較新建立** 或 **手動在結算頁品項層指定**（見第 6 節）  
3. **不可** 同一品項同一結算自動疊加兩個方案  

### 4.3 明細（可多筆）

| 欄位 | 說明 |
|------|------|
| 品項 `product_id` | 必填；MVP 不支援「全品項」 |
| 促銷類型 | 見第 5 節 |
| 參數 | 依類型 |

---

## 5. 促銷類型（分階段）

### 5.1 Phase 1（MVP）

| 類型代碼 | 白話 | 參數 | 計價 |
|----------|------|------|------|
| **FIXED_PRICE** | 期間特價 | `promo_unit_price` | 計價量 × 促銷單價 |
| **DISCOUNT_PCT** | 折扣率 | `discount_pct`（如 90 = 九折） | 計價量 × 池子單價 × 折扣 |
| **BUY_N_GET_M** | 買 N 送 M | `buy_qty`, `free_qty`（如 3, 1） | 見 5.3 |

### 5.2 Phase 2（未做）

- 方案期間 **跨多次結算累計**（促銷餘額表 `promo_progress`）
- 每件減額 `DISCOUNT_AMT`

### 5.3 Phase 3（未做）

- **買 A 送 B**（組合促銷）
- 達量 **事後返利**（月底 AR 批次折讓）

### 5.4 買 N 送 M 計算（Phase 1：僅本次結算單內成組）

以 **買 3 送 1** 為例（一組 4 件，1 件免費）：

```text
settle_qty   = 操作員填寫的「本次賣出量」（全數扣未售池）
bundle_size  = buy_qty + free_qty   // 4
free_qty     = floor(settle_qty / bundle_size) × free_qty
billable_qty = settle_qty - free_qty
```

| 本次結算量 | 贈送 | 計價量 |
|-----------|------|--------|
| 3 | 0 | 3 |
| 4 | 1 | 3 |
| 7 | 1 | 6 |
| 8 | 2 | 6 |

**累計範圍（Phase 1）**：僅 **同一張結算單、同一品項** 加總後計算；不跨歷史結算單累計。

**同品多池列**：結算表可能多列（不同 LOT／單價）。買 N 送 M 在 **品項層** 先加總 `settle_qty`，再算 `free_qty`，免費件分配規則：

| 規則代碼 | 白話 | 定案 |
|----------|------|------|
| **FREE_ON_HIGH_PRICE_FIRST** | 免費件從 **單價較高** 的池列先扣計價量 | **預設**（與收回「高價先退」精神一致） |

分配結果寫入各 `settlement_item` 的 `free_qty` / `billable_qty`（內部分攤）。

## 6. 結算畫面 UI（已定案）

### 6.1 不做

- **不在每一列池子行**「本次結算量」旁放促銷下拉（易漏選、無法表達買三送一跨列、表格過寬）。

### 6.2 要做

#### A. 表頭區（選寄賣案後）

- 摘要框顯示 **本案適用中方案**（唯讀，可連結促銷方案頁）  
- 例：`適用促銷：6月 A 品買三送一、B 品九折`

#### B. 品項池表格

維持一列一筆池子行；欄位調整為：

| 欄位 | 輸入 | 說明 |
|------|------|------|
| 產品／加工廠 Lot | — | 同現行 |
| 出貨／到期 | — | 同現行 |
| 未售剩餘 | — | 同現行 |
| 單位 | — | 同現行 |
| 牌價 | — | 池子 `unit_price`（對照） |
| **本次結算量** | **輸入** | 賣出量；驅動預覽 |
| **贈送** | 唯讀 | 買 N 送 M 等才有值 |
| **計價量** | 唯讀 | 實際計費件數 |
| **小計** | 唯讀 | 該列分攤金額 |

- 促銷名稱：顯示在 **該品項第一列** 小字或 tooltip，不每列下拉。  
- 填寫結算量時 **即時重算** 贈送／計價／小計（僅預覽，送出前仍以後端為準）。

#### C. 衝突時才手選

- 同一品項命中 **多個生效方案** → 在 **該品項區塊**（合併列上方或第一列）出現 **一個** 下拉「套用方案」。  
- 表頭可選 **「本次結算預設方案」**（可選）：整單臨時覆寫，僅當本案有多方案並行時使用。

#### D. 提交

- Confirm 視窗列出：品項、結算量、贈送、計價量、小計、**AR 合計**  
- 後端重新計算並驗證；與預覽不符則拒絕並提示重新整理

### 6.3 手機

- 預覽欄可收合為「展開計價明細」；主輸入仍為本次結算量。

## 7. 結算／AR 資料（擴充）

### 7.1 `consignment_case_settlement_item` 擴充欄位

| 欄位 | 說明 |
|------|------|
| `settle_qty` | 賣出量（扣池） |
| `billable_qty` | 計價量 |
| `free_qty` | 贈送量 |
| `list_unit_price` | 池子牌價（對照） |
| `settle_unit_price` | 實際結算單價 |
| `amount` | `billable_qty × settle_unit_price` |
| `promo_scheme_id` | 套用之方案（可空） |
| `promo_type` | 冗餘快照，便於歷史顯示 |

### 7.2 表頭

- `amount_system` = 明細 `amount` 加總  
- `amount_due` 預設 = `amount_system`（例外調整仍走 AR 模組）

### 7.3 品項池

- `settled_qty` 增加量 = **`settle_qty`**（含贈送件，未售全扣）  
- **`unit_price` 不變**

## 8. 與收回、作廢的關係

| 情境 | 規則 |
|------|------|
| 未售收回 | 與促銷無關 |
| 已售退貨 | **不走收回**；改 **作廢結算**（還原促銷快照）或 **AR 折讓**（見 `ERPv4.2.2` §9.3.1） |
| 作廢結算 | 還原池 `settled_qty`、作廢 AR；促銷套用紀錄隨單作廢 |
| Phase 2 累計促銷 | 作廢須還原 `promo_progress`（後做） |

## 11. 驗收要點（MVP）

1. 建「客戶 A + 品項 X 九折」→ 選該客戶案件結算 → 小計自動為牌價 × 0.9 × 計價量  
2. 建「品項 X 買三送一」→ 單次結算 4 件 → 贈送 1、計價 3、池扣 4  
3. 同品兩池列不同單價、共結算 4 件 → 免費件從 **高單價列** 分攤  
4. 無方案品項 → 贈送 0、計價量 = 結算量、單價 = 牌價  
5. 作廢後池子與 AR 還原；重結算可再套用方案  
6. 結算畫面 **無逐列促銷下拉**（僅衝突時品項層一個下拉）

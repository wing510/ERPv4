## 0. 一句話

**月結統計頁 = 全客戶「請款定案」工作台；第一步只做統計過帳（快照 + 鎖月），不算等級、不算回饋；等級與回饋為可選延伸步驟，皆須統計已過帳後才能做，彼此不強制先後。**

---

## 1. 要解決的問題（現行痛點）

| 痛點 | 說明 |
|------|------|
| 頁名與按鈕語意不符 | 頁面叫「月結統計」，主按鈕卻叫「確認經銷等級」 |
| 第一步職責過重 | 統計、等級累積、鎖月綁在同一顆按鈕 |
| 一般出貨回饋未鎖月 | 僅寄賣強制先統計；一般出貨可直接產回饋，違反「月結」語意 |
| 全客戶 vs 經銷專用混淆 | 列表為全客戶統計，操作卻像經銷等級專頁 |

## 2. 與 v4.2.11 的差異（重要）

| 項目 | v4.2.11（現行） | v4.2.12（本稿） |
|------|----------------|----------------|
| 月結統計過帳 | 快照請款 **+ 寫入等級累積** | **僅**快照請款 + 鎖月 |
| 確認經銷等級 | 與統計同一顆按鈕 | **獨立第二步**，屆時才寫累積 |
| 月結回饋前置 | 有寄賣才須先統計 | **凡要產回饋，一律**須先統計過帳 |
| 沒綁方案客戶 | 列表可見，過帳按鈕多停用 | **可**月結統計過帳 |
| 一鍵結案 | ①+②完成本月結案 | **移除** |

## 3. 三步驟流程（核心定案）

```text
日常請款（寄賣結算／一般出貨過帳）→ 只產 AR
         ↓
【第一步】月結統計過帳 ── 全客戶；有請款即可
         │  快照寄賣／一般／請款淨額；鎖住該客戶該月新請款
         │  不算等級、不算回饋
         ↓
【第二步】確認經銷等級 ── 有綁等級方案才做；須統計已過帳
         │  依方案 stat_source 寫累積；次月待生效
         ↓
【第三步】月結回饋 ── 有綁回饋方案才做；須統計已過帳
            依方案 stat_source 與統計快照算 % → 折讓或次月折抵
```

### 3.1 依賴關係

| 步驟 | 前置條件 | 與其他步驟 |
|------|----------|------------|
| 月結統計過帳 | 當月請款淨額 > 0；尚未有效過帳 | 獨立 |
| 確認經銷等級 | 統計已過帳；有綁等級方案 | **不**強制在回饋之前或之後 |
| 月結回饋 | 統計已過帳；有綁回饋方案；計算基準 > 0 | **不**強制在等級之前或之後 |

### 3.2 沒綁方案客戶（定案）

- **可**執行月結統計過帳（請款定案 + 鎖月）。
- 經銷等級區、月結回饋區 **整塊不顯示**（或僅顯示「未綁定方案」說明，無按鈕）。
- 列表「本月統計」仍顯示狀態（預覽未過帳／已過帳等）。

### 3.3 鎖月口徑（定案）

- **鎖月**綁在「月結統計已過帳」，不綁在回饋或等級。
- 過帳後該客戶該月：**寄賣結算提交**、**一般出貨過帳** 阻擋（與現行 `assertNoPostedMonthlyStatForNewBilling_` 相同，改為純統計觸發）。
- **寄賣、一般皆同** — 解決「僅一般出貨 + 月結回饋」不鎖月漏洞。

## 5. 過帳／作廢規則

### 5.1 月結統計過帳（第一步）

**寫入**

1. 快照當月請款淨額（寄賣／一般／合計）。
2. 寫入 `commercial_dealer_monthly_stat`（CDMS），狀態 POSTED。
3. **不**更新 `dealer_cumulative_amount`。
4. **不**寫入次月待生效等級。
5. **不**產生月結回饋（CDR）。

**護欄**

- 同客戶同月不可重複有效過帳（非 VOID）。
- 請款淨額 ≤ 0 不可過帳。
- 權限：會計／CEO／GA／ADMIN（與現行一致）。

### 5.2 確認經銷等級（第二步）

**寫入**（有綁等級方案時）

1. 讀取該月 **已過帳** CDMS 請款快照。
2. 依等級方案 `stat_source` 計算 `cumulative_add_consignment`、`cumulative_add_general`。
3. 單次更新客戶累積；達標寫入次月待生效。
4. 寫入等級過帳紀錄（資料表見 §7；可沿用 CDMS 擴充或新表）。

**護欄**

- 該月統計須已 POSTED。
- 若統計過帳後又有新請款（drift）→ 須先作廢再重來。
- 同客戶同月不可重複有效等級過帳。

### 5.3 月結回饋（第三步）

**寫入**

1. 該月統計須已 POSTED。
2. 依回饋方案 `stat_source`，以 **統計快照口徑** 計算級距與回饋金額。
3. 寫入 CDR；折讓或次月折抵落地。

**護欄**

- **凡**要產回饋，**一律**須統計已過帳（含僅一般出貨）。
- 同客戶同月不可重複有效回饋。
- 回饋金額 ≤ 0 時依現行規則（不產或擋下，實作時與現行對齊）。

### 5.4 作廢（定案：一鍵代跑 · 甲）

使用者按 **「作廢本月月結」**（或統計紀錄列操作），系統 **單一確認、單一 API**，內部依序：

```text
1. 作廢月結回饋（CDR，若有且 POSTED）
2. 作廢經銷等級（等級過帳紀錄，若有且 POSTED）
3. 作廢月結統計（CDMS，POSTED）
```

- 任一步失敗則整筆 rollback（同一 transaction 或等價 RPC）。
- 作廢後：該月回到預覽未過帳；可補單後重新統計過帳。
- 手動分步作廢：可不實作；若實作，後端仍須擋錯誤順序。

**個別作廢**（討論保留，實作可選）

- 僅作廢回饋、不動統計：允許（補產回饋情境）。
- 僅作廢等級、不動統計：允許。
- 作廢統計：必須透過一鍵代跑（或強制已無有效回饋與等級）。

## 10. 測試矩陣（DEV）

### 10.1 統計過帳

1. 沒綁任何方案 + 有請款 → 可統計過帳；無等級／回饋區。
2. 統計過帳後 → 同月寄賣結算、一般出貨過帳被擋。
3. 列表 [過帳] 與摘要按鈕結果一致。
4. 全部統計過帳：20 位上限、跳過已過帳／無請款／有新單、結果報告正確。

### 10.2 等級（有綁）

5. 統計未過帳 → 確認經銷等級不可按。
6. 統計已過帳 → 可確認等級；累積與次月 pending 正確。
7. 僅一般／僅寄賣／兩者皆有：依 stat_source 計入。

### 10.3 回饋（有綁）

8. 統計未過帳 → 月結回饋不可按（**含僅一般出貨**）。
9. 統計已過帳 → 可先回饋後等級，或先等級後回饋，皆成功。
10. 回饋 stat_source = CONSIGNMENT + 僅一般 → 無回饋可產（基準 0）。

### 10.4 作廢

11. 一鍵作廢：有回饋+等級+統計 → 依序清空；可補單重過帳。
12. 作廢後回饋折讓／折抵餘額還原正確。

### 10.5 漂移

13. 統計已過帳後若因例外可補單（不應發生）或作廢後補單 → 重過帳後數字正確。
14. 已過帳・有新單：橘標、不可再過帳直至作廢。

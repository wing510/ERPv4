## ERP 主檔 — 版型 A（列表上／表單下）討論定案說明書

> **狀態**：討論定案（**尚未開始改程式**）  
> **日期**：2026-06-16  
> **範圍**：MASTER DATA 主檔五頁 UI 版型統一  
> **不動**：Company 公司設定（維持整頁單一表單、無列表）

---

## 0. 一句話總結

**主檔日常是「先搜尋、Load、改一兩欄」→ 列表放上面、編輯表單放下面；列表固定高度內捲、不撐爆頁面；Load 後自動捲到表單。**

---

## 1. 背景（為什麼要改）

### 現況（五頁主檔）

| 模組 | 檔案 | 現在版面 |
|------|------|----------|
| Products 產品 | `modules/products.html` | **表單上** → 列表下 |
| Suppliers 供應商 | `modules/suppliers.html` | 同上 |
| Customers 客戶 | `modules/customers.html` | 同上 |
| Warehouses 倉庫 | `modules/warehouses.html` | 同上 |
| Users 使用者 | `modules/users.html` | 同上 |

進頁第一眼是**空白編輯表單**，要往下捲才看到列表，與實際用法（先找既有資料）不一致。

### 曾考慮的疑慮

「列表在上、表單在下 → 列表很多時，Load 要捲很下面？」

**澄清**：主檔列表外層已有 `table-wrapper-master`（`index.html`），高度約 **min 480px～70vh**，**列數再多也是表格內部捲動**，不會把整頁無限撐長。表單距離列表底部約 **半屏～一屏**，可接受。

### 定案選項

| 選項 | 說明 | 結論 |
|------|------|------|
| **A（採用）** | 列表上 + 表單下 + Load 自動捲表單 | ✅ |
| B | 左右分欄（寬螢幕列表左、表單右） | 暫不做（實作成本高；日後若常「邊看列表邊改」再評估） |
| 維持現狀 | 表單上、列表下 | ❌ |


**進頁預期**：使用者先看到搜尋與列表；表單在下方待命。

## 4. 互動規則（定案）

### 4.1 列表區

- 維持 **`table-wrapper table-wrapper-master`**，不改成全頁無限列表。
- 搜尋、排序、篩選行為**不變**。
- 列表每列 **Load** 按鈕行為不變（帶入表單欄位）。

### 4.2 Load → 自動捲到表單

- 點列表 **Load** 後：  
  1. 帶入下方表單（與現相同）  
  2. **自動捲動至表單區**（smooth scroll）

**實作注意**：

- 現有 `scrollToEditorTop()`（`js/core/utils.js`）是捲到 `#content` **頂部**，適合「表單在上」；改版後需改為捲到**表單卡片**（建議新增共用函式，例如 `scrollToMasterForm_(formCardId)`，或各頁統一 `id="*_edit_card"`）。
- 滾動容器為 `#content`（非 `window`），與現有 ERP 一致。

### 4.3 建立／更新／清除

- 按鈕語意與 API **不變**。
- **建立成功**：可維持現有行為（清空表單、刷新列表）；是否捲回列表頂為**可選**，實作時與 Products 範本一併確認即可。
- **更新成功**：建議留在表單區（已捲在表單附近），僅刷新列表資料。

### 4.4 新增表單（可選強化）

比照 **Case 案件管理**「開新寄賣案」：

- 表單區可**預設收合**，按「新增 ○○」再展開。  
- **v1 實作可不做收合**，僅調整上下順序 + Load 捲動；收合列為 Phase 2。

## 5. 與 Case 案件管理的一致性

| 項目 | Case 案件管理（已上線） | 主檔版型 A |
|------|------------------------|------------|
| 列表位置 | 上 | 上 |
| 明細／編輯 | 下（點明細展開） | 下（表單常駐） |
| 進頁自動開明細 | 否（已移除 session 自動開案） | 否 |
| 列表高度 | 一般表格 | **固定高度** `table-wrapper-master` |

主檔與 Case 精神一致：**先查資料，再操作**；主檔多一層「列表固定高度」避免表單被擠到視窗外太遠。

## 6. 實作順序（建議）

1. **Products 產品** — 當範本（欄位中等、`js/products.js` 相對單純）  
2. 驗收後依序：**Suppliers → Customers → Warehouses → Users**  
3. 五頁完成後：更新 `js/help.js` 對應主檔說明（若有「表單在上」舊文案）  
4. **不修改** `company-settings` 與後端 API

### 每頁預估異動

| 類型 | 路徑 | 內容 |
|------|------|------|
| HTML | `modules/products.html` 等 | 兩張 card **對調順序**（列表 card 在上） |
| JS | `js/products.js` 等 | Load／建立後改呼叫捲到表單；必要時加 `edit_card` id |
| CSS | 多半沿用 | `table-wrapper-master` 已存在，無需新樣式除非微調間距 |
| 共用 | `js/core/utils.js`（建議） | 新增 `scrollToMasterForm_` 供五頁共用 |

## 7. 驗收清單（實作後）

- [ ] 進入 Products（及其他四頁）：**第一眼為搜尋 + 列表**，不是空白表單。  
- [ ] 列表列數很多時：僅**表格局部捲動**，頁面長度無異常。  
- [ ] 點 **Load**：欄位正確帶入 + **自動捲到表單**（不需手動找表單）。  
- [ ] **建立／更新／清除** 功能與改版前一致。  
- [ ] **Company 公司設定** 版面未變。  
- [ ] Ctrl+F5 強制重新整理後行為正常。  
- [ ] 窄螢幕（`table-wrapper-master` 約 42vh）仍可操作。

# ERP 規則清單（主檔｜v4.1）

> 目的：把主檔規則集中成「單一清單」，避免規則散落在各頁造成改一邊漏一邊。  
> 範圍：產品 / 供應商 / 客戶 / 倉庫 / 使用者。  
> 原則：只寫「規則與口徑」，不寫程式細節；文字以現場看得懂為主。

**v4.1 環境**
- 資料存放：**Supabase**（PostgreSQL 表）
- API：**Node**（本機 DEV `http://127.0.0.1:1314/exec`）
- 使用者表名：`erp_user`（畫面仍叫「使用者」；因 `user` 為資料庫保留字）
- 後台查 raw 資料：各模組 **Supabase** 按鈕 → Table Editor（CEO / 總務 / ADMIN）

---

### A. 主檔共通規則（所有主檔都適用）

- **必填原則**
  - **ID、名稱**為必填（沒填就不能建立/更新）。
  - 若有「分類/類型/國家」等欄位選到「其他」，**必須補充原因/備註**（避免資料品質崩壞）。

- **狀態（使用中/停用）**
  - **停用後**：不可再被新單據選用，但**歷史紀錄必須保留**。
  - **停用前提醒**：若該主檔已被交易資料使用，應提示「停用影響」並要求再次確認。
  - **權限限制（主檔狀態）**：狀態變更屬高風險，僅允許指定主管角色操作（避免助理誤停用造成流程中斷）。

- **建立/更新的紀錄口徑**
  - 建立：要能回溯「誰建立、何時建立」（`created_by` / `created_at`）。
  - 更新：要能回溯「誰更新、何時更新」（`updated_by` / `updated_at`）。

- **刪除口徑（v4.1）**
  - 主檔採**軟刪/停用**，不物理刪除；避免追溯斷鏈。

- **錯誤訊息口徑（主檔通用）**
  - **必填缺漏**：明確指出「缺什麼」與「在哪裡補」。
  - **權限不足**：明確指出「目前帳號沒有權限」與「請找誰處理/去哪裡開權限」。
  - **資料不存在**：用「找不到資料」而不是讓畫面默默失敗。
  - **後端未就緒**：若 API 未啟動或 session 失效，提示重新登入（API 重啟後 session 會失效）。

---

### B. 產品（Products｜表 `product`）

- **必填**
  - 產品編號、產品名稱
- **建議一致性**
  - 產品名稱/規格的顯示規則要全站一致（下拉/列表/追溯）。
- **停用**
  - 若產品已在批次、異動、採購/進口、加工、出貨、銷售等流程出現過，停用前需提示影響並再次確認。

---

### C. 供應商（Suppliers｜表 `supplier`）

- **必填**
  - 供應商編號、供應商名稱
- **「其他」必填原因**
  - 國家或供應商類型選「其他」時，原因/備註必填。
- **停用**
  - 若供應商已在採購、進口、加工等流程出現過，停用前需提示影響並再次確認。

---

### D. 客戶（Customers｜表 `customer`）

- **必填**
  - 客戶編號、客戶名稱
- **「其他」必填原因**
  - 分類或國家選「其他」時，原因/備註必填。
- **停用**
  - 若客戶已在銷售、出貨等流程出現過，停用前需提示影響並再次確認。

---

### E. 倉庫（Warehouses｜表 `warehouse`）

- **必填**
  - 倉庫編號、倉庫名稱、類別（溫層等）
- **一致性**
  - 倉庫屬於核心主檔：收貨、批次、庫存異動、出貨、加工等流程都會引用，欄位定義與顯示需一致。
- **停用**
  - 停用前需提示：停用後不可再作為新單據/新批次的倉庫選項，但歷史紀錄保留。

---

### F. 使用者（Users｜表 `erp_user`）

- **必填**
  - 使用者帳號、角色、狀態
- **登入（v4.1）**
  - 支援 **admin 密碼登入** 與 **Google 登入**（以 `email` 對應 `erp_user`）。
- **權限配置（白話口徑）**
  - 「能看到/能進入的功能」由管理者配置；未配置就不應顯示入口，也不可透過網址直接進入。
- **停用**
  - 停用後不得再登入；歷史操作紀錄（`logs`）仍要能查到操作者。

---

### G. 對外溝通建議（給現場/主管）

- **主檔停用**：停用是「管理新資料的入口」，不是刪除；停用後歷史仍在，但新單據選不到。
- **權限不足**：先確認「這個人是否該做這件事」，再由管理者調整權限或由主管代操作。
- **資料在哪**：v4.1 正式資料在 Supabase，不在 Google 試算表；舊 Sheet 若有保留僅作歷史參考，不會即時同步。
